Categories

Metadata content

These are the elements that reside in head section of doc, such as <title>, <script>, <style>. These elements contain the doc metadata, which is info about the doc itself, such as how to present the document or what other documents are related to this one, such as style sheets.

Flow content

They are used within the <body> section, such as <p>, <footer>.

Sectioning root

Content on web page is divided into sections. <body> is the root or parent element of all content sections, making it a sectioning root. Other elements in this category include <blockquote> and <td>.

Sectioning content

Sectioning content divides a document into sections, each of which can have its own set of headings. These elements group sections of content on the page. Elements in this category include <section>,<article> and <nav>. Sectioning elements are supported by all modern browsers including mobile browsers.

Heading content

This category includes the heading elements, <h1> through to <h6>, plus the new <header> element for creating heading sections

Phrasing content

Phrasing content includes elements that are used within lines of text in documents. These include<em>, <strong> and <span>. They were called inline elements in HTML 4.0.

Embedded content

Embedded content elements load external content into the web page. This content can be image, video or audio files. The <img> element as well as the new <audio> and <video>elements are part of this category

Interactive content

Interactive elements let users interact with the web page content. Interactivity depends on the users browser and input device, such as mouse, keyboard, touchscreen or voice input. Elements in this category include the <a>element, which creates clickable hyperlinks, plus the <audio> and <video>elements if the user can control the content such as by using play or pause buttons. Flash animations and other content types such as presentations or web based learning also accept user interaction. Form controls are also part of this category.

Transparent

Some elements have transparent content models, which means that their allowed content is inherited form their parent element. They may contain any content that their parent element may contain, in addition to any other allowances or exceptions described for the element. For example the<a> element often occurs within a <p> element parent and will usually follow the content rules of its parent element
